helix.yaml – Helix Project Manifest

The official configuration file for all Helix-powered MQL5/MQL4 projects.

Fields

FieldRequiredTypeDefaultDescription
nameYesstr——
versionYesstr—Required SemVer version
descriptionNoOptionalNone—
authorNoOptionalNone—
licenseNoOptionalMIT—
typeYesMQLProjectType—expert, indicator, library, include, etc.
targetNoMQLTargetMQL5MQL4 or MQL5
dependenciesNoobject{}—
include_modeNoIncludeModeincludeBuild preparation mode: 'include' (default) or 'flat'
entrypointsNo*OptionalNone.mq4/.mq5 file list. Required except for type='include'
distNoOptionalNone—
helixNoOptionalNone—

Project Types (type)

  • expert → Expert Advisor
  • indicator → Custom indicator
  • script → Script
  • library → Reusable library
  • include → Only .mqh headers (no entrypoints allowed)

Dependency Formats (supported)

  • https://github.com/user/repo.git#v1.2.3 → Git + version/tag/branch/commit
  • file://../my-lib → Local Helix project
  • ./libs/utils or ../common → Relative path

Full Example


name: QuantumEA
version: 3.1.0
description: Advanced multi-timeframe Expert Advisor
author: John Trader
license: MIT
type: expert
target: MQL5
include_mode: flat

entrypoints:
  - Strategy.mqh
  - QuantumEA.mq5

dependencies:
  bars: https://github.com/trader/bars.git#v2.1.0
  utils: file://../common/utils
  indicators: ./local/indicators

dist:
  dist:
    - id: release
      name: QuantumEA-${version}.zip
      items:
        - dependencyId: this
          src: QuantumEA.mq5
          dst: Experts/QuantumEA.mq5
